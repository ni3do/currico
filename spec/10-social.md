# Follow Seller & Notifications

## Implementation Status

**Status: UI Only**

- Follow button UI: Implemented
- Updates/Following page UI: Implemented
- Follow database model: Not implemented
- Follow API endpoints: Not implemented
- Notification system: Not implemented

---

## Overview

Allow users to follow sellers and receive updates about new content.

## Follow Feature

### Interaction Points
- "Follow" button on seller profile pages
- "Follow" button on resource detail pages

### Button Design
- Subtle but discoverable
- Toggle between Follow/Unfollow states

## Updates Section

### Location
- "Updates" or "Following" section in user area

### Content Display
- New resources and bundles from followed sellers
- Reverse chronological order
- Straightforward vertical list

### Item Information (Minimal)
- Item title
- Seller name
- Date published
- "View Resource" link (clearly styled)

## Notification Philosophy

- Intentionally low-noise
- No cluttered badges
- Only necessary notification types for MVP
- No complex notification preferences

## UI Guidelines

- Follow/unfollow toggles remain subtle but discoverable
- Primary links ("View Resource") are clearly styled
- List format prioritizes readability over density
